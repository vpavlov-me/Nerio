## API migration

- `Badge.variant` is deprecated in favour of `Badge.tone`. Existing `neutral`, `info`, `success`, and `danger` values keep working. New code may also use `accent` and `warning` through `tone`.
- `Button.subtle` is deprecated in favour of `secondary`; `Button.destructive` is deprecated in favour of `danger`. Both aliases normalize to canonical variants until the next major.
- `IconButton` is a deprecated compatibility wrapper. New code should use `<Button icon={Settings} aria-label="Settings" />`; remove the alias in the next major.
- `Button.loadingLabel` has no accessible effect and is scheduled for removal in the next major. Loading preserves the visible action name instead.
- Use `Card href="…"` for a full-card destination; linked Cards cannot choose a surface `as` root and must not contain nested interactive descendants.
- `CardVisual` accepts generic visual content such as an icon, avatar, logo, preview, or media. Its `placement` is deliberately limited to `inset` or `bleed`; `CardAction` belongs only in surface Cards.
- Button shortcut display is supplemental. Add `aria-keyshortcuts` yourself when the shortcut is functional, for example `aria-keyshortcuts="Meta+K"`.
