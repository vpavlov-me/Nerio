# Core 1.0 platform primitive coverage

## Status and purpose

This document is the canonical Core 1.0 decision for web-platform primitive coverage. It defines
whether Nerio covers a capability with a dedicated component, supported native composition, native
HTML guidance, or an intentional deferral. It does not require a React wrapper for every HTML
element.

`data/component-catalog.json` contains the machine-readable projection of the decision. The catalog
validator compares that projection with the marked matrix below so component inventory and coverage
claims cannot drift independently. Other documents should link here instead of copying the matrix.

## Product rule

A release-relevant platform capability is covered when one of these modes applies:

1. `dedicated-component`: shared behavior, accessibility, styling, state, or composition creates a
   durable Core responsibility.
2. `supported-native`: the native element or state is intentionally supported with existing Nerio
   foundations and tokens while the browser keeps its semantics and behavior.
3. `native-guidance`: plain semantic HTML is the preferred public contract because a wrapper would
   add no stable product value.
4. `intentionally-deferred`: the capability is a product workflow, advanced composition, or
   non-essential abstraction outside Core 1.0.

Core owns reusable semantics, accessibility contracts, component behavior, public types, and
tokenized presentation. Consumers own data, validation policy, persistence, business rules,
localization copy, network operations, and workflow orchestration. Pro owns reusable product-ready
workflows. Native controls keep browser-owned picker chrome, form behavior, value semantics, and
platform integration unless a dedicated Core component explicitly replaces that presentation.

## Approved 1.0 delivery sequence

- [#258](https://github.com/vpavlov-me/Nerio/issues/258) provides a supported native Input path for date,
  month, week, time, and `datetime-local` controls.
- [#259](https://github.com/vpavlov-me/Nerio/issues/259) adds a single-value Slider backed by Base
  UI. Multi-thumb ranges remain outside Core 1.0.
- [#260](https://github.com/vpavlov-me/Nerio/issues/260) adds native file selection without upload
  workflow ownership.
- [#261](https://github.com/vpavlov-me/Nerio/issues/261) adds a localizable single-date Calendar.
- [#262](https://github.com/vpavlov-me/Nerio/issues/262) composes a bounded single-date DatePicker
  from the accepted temporal, Calendar, and overlay contracts.

The Calendar and DatePicker are deliberate bounded exceptions to the native-wrapper rule because
single-date selection is universal and a custom calendar has a durable accessibility contract. The
implementation tasks may refine their APIs, entrypoints, and internal composition, but they must not
change the responsibility boundary above without updating this decision and the roadmap.

## Validation and change policy

Run `pnpm test:catalog` and `pnpm validate:catalog` after changing this document, the platform
coverage projection, or any linked catalog component. A new 1.0 blocker requires a focused GitHub
issue and roadmap entry before its matrix status can become `planned`. Removing a covered capability
or expanding a deferred workflow into Core requires an explicit product decision and synchronized
updates to this document, `data/component-catalog.json`, `COMPONENTS.md`, agent guidance, and public
AI guidance.
