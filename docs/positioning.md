# Positioning

Nerio is an open-source, source-first design system for modern digital products.

It is universal by design. The system should feel credible in SaaS applications, consumer products, marketplaces, dashboards, internal tools, content platforms, creator tools, productivity products, and data-rich workflows. Industries are examples of use, not restrictions on the system.

## What Nerio is

- A React and TypeScript component foundation.
- A semantic token system with runtime themes and density modes.
- A source registry and CLI for installing editable component code.
- AI-readable component metadata and MCP discovery tooling.
- A calm, precise visual language that can be branded through tokens.

## What Nerio is not

- Not a fintech, crypto, banking, or SaaS-only system.
- Not a shadcn/ui distribution or registry.
- Not a wrapper around Radix UI, Headless UI, Ariakit, or another overlapping primitive layer.
- Not a marketplace, hosted service, or template catalog.

## Principles

- APIs are generic and domain-agnostic.
- Tokens describe visual and interaction intent rather than industry-specific concepts.
- Documentation uses broadly applicable examples by default.
- Themes support neutral foundations and brand expression.
- Showcase applications demonstrate complexity without defining product scope.

## Voice

Documentation should be clear, practical, and product-minded. Examples should prefer universal scenarios such as projects, collections, content, collaboration, settings, search, activity, and analytics.
