# Positioning

Nerio is an open-source, source-first design system for modern digital products.

It is universal by design. The system should feel credible in SaaS applications, consumer products, marketplaces, dashboards, internal tools, content platforms, creator tools, productivity products, AI interfaces, and data-rich workflows. Industries are examples of use, not restrictions on the system.

## Product model

Nerio has two product layers:

- **Nerio Core**: the open-source foundation for tokens, themes, primitive and base UI components, public documentation, public registry, CLI, and public MCP/component discovery.
- **Nerio Pro**: the future paid layer for advanced product components, templates, premium themes, Figma assets, advanced registry items, and Pro MCP/AI tooling.

Core = building blocks. Pro = product-ready solutions.

## What Nerio is

- A React and TypeScript component foundation.
- A semantic token system with runtime themes and density modes.
- A source registry and CLI for installing editable component code.
- AI-readable component metadata and MCP discovery tooling.
- A calm, precise visual language that can be branded through tokens.
- A system with a clear path for Pro components and templates without weakening the open-source Core.

## What Nerio is not

- Not a fintech, crypto, banking, or SaaS-only system.
- Not a shadcn/ui distribution or registry.
- Not a wrapper around Radix UI, Headless UI, Ariakit, or another overlapping primitive layer.
- Core is not a marketplace, hosted service, or template catalog.

## Principles

- Core APIs are generic and domain-agnostic.
- Pro patterns may be domain-specific when they provide clear product value.
- Tokens describe visual and interaction intent rather than industry-specific concepts.
- Documentation uses broadly applicable examples by default.
- Themes support neutral foundations and brand expression.
- Showcase applications demonstrate complexity without defining product scope.

## Voice

Documentation should be clear, practical, and product-minded. Core examples should prefer universal scenarios such as projects, collections, content, collaboration, settings, search, activity, and analytics. Pro examples may use more specific SaaS, fintech, crypto, dashboard, billing, and AI scenarios.
